# Security Policy

## Supported Versions

Security updates are currently applied to the latest `main` branch.

## Reporting a Vulnerability

Please do not open public issues for suspected vulnerabilities.

Report security concerns privately by emailing:

- `mike.moulder.dev@gmail.com`

Include as much detail as possible:

- Affected component/file
- Reproduction steps
- Impact assessment
- Any proof-of-concept details

You can expect an initial response within 5 business days.

## Disclosure Process

1. We acknowledge receipt and begin triage.
2. We validate and assess severity.
3. We develop and test a fix.
4. We coordinate disclosure timing with the reporter.
5. We publish a patch and advisory when appropriate.
