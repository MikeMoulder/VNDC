# Contributing

Thanks for your interest in contributing to OpenGradient.

## Getting Started

1. Fork the repository and create a feature branch from `main`.
2. Set up the backend:
   - Create and activate a virtual environment.
   - Install dependencies from `backend/requirements.txt`.
   - Copy `backend/.env.example` to `backend/.env` and set required values.
3. Set up the frontend:
   - Install dependencies in `frontend/`.
   - Copy `frontend/.env.example` to `frontend/.env.local`.

## Development Workflow

1. Keep changes focused and small.
2. Write clear commit messages.
3. Update documentation when behavior or setup changes.
4. Ensure the project runs locally before opening a pull request.

## Pull Requests

1. Open a pull request against `main`.
2. Include:
   - What changed
   - Why it changed
   - How it was tested
3. Link any related issue.

## Code Style

- Python: follow PEP 8 conventions.
- TypeScript/React: keep components typed and readable.
- Prefer descriptive names over short abbreviations.

## Reporting Bugs

When opening an issue, include:

- Steps to reproduce
- Expected behavior
- Actual behavior
- Relevant logs/screenshots
- Environment (OS, Python version, Node version)

## Scope and Safety

This project provides informational outputs only and is not financial advice.
Contributions that weaken output safety, transparency, or verifiability may be declined.
